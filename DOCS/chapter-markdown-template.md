# 章节 Markdown 编写规范（模板）

> 说明：程序会在“Markdown 扫描目录”中扫描所有 `.md/.markdown` 文件。通常这个目录就是当前播放列表中本地视频所在目录。

## 1) 最小可用模板

```markdown
# 视频标题（可选）

<card>
## 章节1 #产品/功能 #新手

<time datetime="00:00:10">开场</time>
<time datetime="00:01:25">功能总览</time>
</card>

<card>
## 章节2 #产品/进阶 #实战

<time datetime="00:05:40">实战演示</time>
<time datetime="00:08:20" video="另一个视频.mp4">跳转到另一个视频</time>
</card>
```

---

## 2) `<time>` 标签规则

- 必填属性：`datetime="HH:MM:SS"`（例如 `00:00:40`）。
- 可选属性：`video="文件名.mp4"`。
- `video` 仅支持**文件名**，不要写路径（不能包含 `/` 或 `\\`）。
- 点击 `<time>` 渲染后的链接时：
  1. 程序先在播放列表里按文件名查找目标视频；
  2. 如果没找到，再到“Markdown 扫描目录”按文件名查找并自动加入当前播放列表后播放。

---

## 3) `#话题` 索引规则（章节话题索引面板）

- 话题只能在 `<card> ... </card>` 块内被索引。
- 话题格式：`#一级` 或 `#一级/二级/三级`。
- 支持字符：中文、英文、数字、下划线 `_`、短横线 `-`。
- 一个 `<card>` 内可写多个话题，例如：`#AI/入门 #AI/提示词`。
- 一个 `<card>` 内的每个 `<time>` 会同时挂到该卡片下的所有话题路径中。

---

## 4) 常见错误

- `<time>` 缺少 `datetime`。
- `datetime` 不是 `HH:MM:SS`。
- `<time>` 标签未闭合。
- `video` 写成了完整路径而不是文件名。
